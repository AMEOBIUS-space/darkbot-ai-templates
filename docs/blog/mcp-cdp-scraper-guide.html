Building a Zero-Dependency MCP Scraping Server in Pure Python

July 6, 2026 · 12 min read · AMEOBIUS

MCP Chrome DevTools Protocol Web Scraping Python Zero Dependencies AI Agents
>

Why Zero Dependencies?

Most MCP scraping servers wrap Playwright or Puppeteer — heavy dependencies that add 200MB+ to your Docker image, require npm, and break in headless environments. What if you could give your AI agent 12 scraping tools with nothing but Python's standard library?

That's what mcp-cdp-scraperdoes. It connects to any Chrome/Chromium instance via raw WebSocket CDP commands — no Playwright, no Selenium, no Puppeteer, no pip installs beyond Python itself.

>

Architecture

MCP Client (Claude / Hermes / any)

 ▼
MCPCDPScraperServer (src/server.py)

 ▼
CDPScraper (src/cdp_scrape.py)
 │ Raw WebSocket CDP commands
 ▼
Chrome/Chromium (--remote-debugging-port=9222)

Web Page → Structured Data

The server exposes 12 MCP tools. Each tool call is dispatched to the CDPScraperengine, which sends raw CDP commands over a WebSocket connection to a running Chrome instance.

3. The 12 MCP Tools

>

scrape_page — Full Page Extraction

Navigate to a URL and extract everything in one call: title, meta tags, headings (h1-h3), paragraphs, lists, links, and images.

from src.server import MCPCDPScraperServer
import json

server = MCPCDPScraperServer()
result = server.handle_tool_call("scrape_page", {
 "url": "https://news.ycombinator.com",
 "wait": 2.0
})
data = json.loads(result)
print(data["structured"]["title"]) # "Hacker News"
print(f"Found {data['link_count']} links")
print(f"Found {data['image_count']} images")
>

extract_text — Targeted Text Extraction

result = server.handle_tool_call("extract_text", {
 "selector": "article body"
})
>

extract_links — All Hyperlinks

Returns a list of {text, href, target}objects. Optionally resolves relative URLs against a base URL.

result = server.handle_tool_call("extract_links", {
 "base_url": "https://example.com"
})
>

extract_images — All Images

Returns {src, alt, width, height}for every <img>on the page.

>

extract_table — Table to 2D Array

result = server.handle_tool_call("extract_table", {
 "selector": "table.data-table"
})
# Returns: [["Name", "Price"], ["Widget", "$9.99"], ...]
>

fill_form — React/Vue Compatible

This is the killer feature. Most scraping tools set element.value = "x"directly, which React and Vue ignore because they track state through synthetic events. Our fill_formuses native setters and dispatches input+ changeevents:

result = server.handle_tool_call("fill_form", {
 "fields": {
 "#email": "user@example.com",
 "#password": "secret123"
 },
 "submit": "#login-button"
})

The JavaScript under the hood:

const setter = Object.getOwnPropertyDescriptor(
 HTMLInputElement.prototype, "value"
).set
setter.call(el, "value")
el.dispatchEvent(new Event("input", {bubbles: true}))
el.dispatchEvent(new Event("change", {bubbles: true}))
>

click_element — Click by Selector

result = server.handle_tool_call("click_element", {
 "selector": "#submit-btn"
})
>

screenshot — Capture or Save

result = server.handle_tool_call("screenshot", {
 "save_path": "/tmp/page.png"
})
>

get_html — Element Outer HTML

result = server.handle_tool_call("get_html", {
 "selector": "main"
})
>

wait_for — Poll for Element

Waits for a CSS selector to appear. Polls every 500ms up to a configurable timeout. Essential for SPAs with dynamic content.

result = server.handle_tool_call("wait_for", {
 "selector": ".results-loaded",
 "timeout": 15
})
>

scroll_to — Element or Coordinates

# Scroll to element
server.handle_tool_call("scroll_to", {"selector": "#footer"})

# Scroll to coordinates
server.handle_tool_call("scroll_to", {"x": 0, "y": 2000})
>

list_tabs — Browser Tab Management

result = server.handle_tool_call("list_tabs", {})
# Returns: {"tabs": [{"title": "...", "url": "...", "id": "..."}]}
>

4. Integration with Claude Desktop

Add this to your claude_desktop_config.json:

 "command": "python",
 "args": ["-m", "src.server", "--stdio"],

7. Raw WebSocket CDP — No Libraries

The CDPConnectionclass implements the WebSocket protocol from scratch using only socket, struct, and base64:

class CDPConnection:
 def connect(self):
 host, port, path = self._parse_ws_url()
 self.sock = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
 self.sock.connect((host, port))
 # WebSocket handshake (RFC 6455)
 key = base64.b64encode(os.urandom(16)).decode()
 handshake = f"GET {path} HTTP/1.1\r\n..."
 self.sock.sendall(handshake.encode())

 def send_command(self, method: str, params: dict) ->dict:
 self.msg_id += 1
 payload = json.dumps({
 "id": self.msg_id,
 "method": method,
 "params": params
 })
 self._send_ws_frame(payload)
 return self._recv_response()

No websocketspip package. No websocket-client. Just raw bytes over a TCP socket with RFC 6455 framing.

Why This Matters for AI Agents

The Model Context Protocol is becoming the standard interface for AI agents to interact with external tools. But most MCP servers for web scraping are heavy — they pull in Playwright (190MB), require npm, and add complexity to your deployment.

With mcp-cdp-scraper, your agent gets production-grade scraping capabilities with:

  • Zero pip dependencies— just Python 3.10+ and a Chrome binary
  • 12 well-defined tools— structured I/O that LLMs can reason about
  • React/Vue form filling— the hardest part of browser automation, solved
  • STDIO JSON-RPC— works with any MCP-compatible client
  • 38 tests— production-ready, not a toy
