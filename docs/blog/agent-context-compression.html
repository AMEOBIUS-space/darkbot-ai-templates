Practical Context Window Compression for AI Agents

>

MCPPythonPerformance

>

2026-07-09

>

Every tool call your agent makes costs context window tokens. Read a file? Tokens. Run a search? Tokens. List a directory? More tokens. After 20 tool calls, your agent has burned through 15,000 tokens of context on tool output alone — tokens that should be reserved for reasoning.

>

This article shows five compression strategies used in mcp-context-guard, each implementable in under 50 lines of pure Python.

>

Strategy 1: Head-Tail Truncation

>

The simplest and most effective technique. Most tool outputs have a "useful prefix" (the answer) and a "useful suffix" (the summary), with noise in the middle.

>
def head_tail_truncate(text: str, max_tokens: int = 500) ->str:
 """Keep the first and last N characters, drop the middle."""
 max_chars = max_tokens * 4 # rough token-to-char estimate
 if len(text) <= max_chars:
 return text
 
 head_size = int(max_chars * 0.6) # 60% from head
 tail_size = int(max_chars * 0.3) # 30% from tail
 # 10% for the truncation marker
 
 head = text[:head_size]
 tail = text[-tail_size:]
 omitted = len(text) - head_size - tail_size
 
 return f"{head}\n\n... [{omitted:,} chars omitted] ...\n\n{tail}"
>
Always include the omission count. The agent needs to know it's seeing partial output to decide whether to request more.
>

Strategy 2: JSON Key Flattening

>

Tool outputs are often deeply nested JSON. Flatten the keys and drop null values:

>
def flatten_json(obj: dict, prefix: str = "") ->dict:
 """Flatten nested dict to dotted keys, drop None values."""
 result = {}
 for key, value in obj.items():
 full_key = f"{prefix}.{key}" if prefix else key
 if value is None:
 continue
 elif isinstance(value, dict):
 result.update(flatten_json(value, full_key))
 elif isinstance(value, list):
 if value:
 result[full_key] = f"[{len(value)} items]"
 if len(value) <= 3:
 for i, item in enumerate(value):
 if isinstance(item, dict):
 result.update(flatten_json(item, f"{full_key}[{i}]"))
 else:
 result[f"{full_key}[{i}]"] = item
 else:
 result[full_key] = value
 return result
>

Before: 340 tokens of nested JSON. After: 120 tokens of flat key-value pairs.

>

Strategy 3: Content Deduplication

>

When an agent reads the same file twice, or searches for patterns that overlap, you get duplicate content. Track seen hashes and replace with a back-reference:

>
import hashlib

class Deduplicator:
 def __init__(self):
 self.seen: dict[str, int] = {} # hash ->call_number
 
 def deduplicate(self, text: str, call_num: int) ->str:
 # Hash each line, track unique content
 lines = text.split("\n")
 output = []
 for line in lines:
 h = hashlib.md5(line.strip().encode()).hexdigest()[:8]
 if h in self.seen and line.strip():
 first_seen = self.seen[h]
 output.append(f"[↻ see call #{first_seen}]")
 else:
 self.seen[h] = call_num
 output.append(line)
 return "\n".join(output)
>

Strategy 4: Relevance Filtering

>

When the agent is debugging an import error, it does not need 200 lines of CSS. Filter tool output by keyword relevance:

>
def relevance_filter(text: str, query: str, context_lines: int = 3) ->str:
 """Keep only lines matching the query plus surrounding context."""
 lines = text.split("\n")
 query_terms = set(query.lower().split())
 keep = set()
 
 for i, line in enumerate(lines):
 line_lower = line.lower()
 if any(term in line_lower for term in query_terms):
 # Keep this line + N lines of context
 for j in range(max(0, i - context_lines), min(len(lines), i + context_lines + 1)):
 keep.add(j)
 
 if not keep:
 # No matches — return first 10 lines as fallback
 keep = set(range(min(10, len(lines))))
 
 result = []
 prev = -1
 for i in sorted(keep):
 if prev >= 0 and i >prev + 1:
 result.append("...")
 result.append(lines[i])
 prev = i
 
 return "\n".join(result)
>

Strategy 5: Semantic Bucketing

>

Group related lines and summarize each bucket. Perfect for log output and stack traces:

>
def bucket_output(text: str, patterns: dict) ->str:
 """Group lines matching patterns, summarize counts."""
 buckets = {name: [] for name in patterns}
 buckets["_other"] = []
 
 for line in text.split("\n"):
 matched = False
 for name, pattern in patterns.items():
 if pattern.lower() in line.lower():
 buckets[name].append(line)
 matched = True
 break
 if not matched:
 buckets["_other"].append(line)
 
 result = []
 for name, lines in buckets.items():
 if not lines:
 continue
 if len(lines) <= 2:
 result.extend(lines)
 else:
 result.append(f"### {name} ({len(lines)} lines)")
 result.extend(lines[:2])
 if len(lines) >2:
 result.append(f"... {len(lines) - 2} more")
 
 return "\n".join(result)

# Example patterns for terminal output:
patterns = {
 "Errors": "error",
 "Warnings": "warning", 
 "Files": "files changed",
 "Tests": "passed",
}
>

Putting It All Together

>

Chain these strategies in order of aggressiveness:

>
def compress_tool_output(
 text: str, 
 query: str = "",
 max_tokens: int = 500,
 deduplicator: Deduplicator = None,
 call_num: int = 0
) ->str:
 # 1. Deduplicate (cheapest, highest impact on repeated calls)
 if deduplicator:
 text = deduplicator.deduplicate(text, call_num)
 
 # 2. Relevance filter (if query provided)
 if query:
 text = relevance_filter(text, query)
 
 # 3. Bucket (for long structured output)
 if len(text) >2000:
 text = bucket_output(text, {
 "Errors": "error",
 "Warnings": "warning",
 })
 
 # 4. Head-tail truncate (last resort)
 text = head_tail_truncate(text, max_tokens)
 
 return text
>
Always inform the agent when output is compressed. Add a header like [COMPRESSED: 3400→480 tokens, 85% reduction]so the agent knows to request full output if needed.
