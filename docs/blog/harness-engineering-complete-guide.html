How They Work Together

>

1. Request arrives

>

The Skill Routeranalyzes the task and loads only relevant tools (84% fewer tools in the prompt).

>

2. Before each tool call

>

The Permission Guardchecks if the call is safe. rm -rfis denied. lsis allowed. Unknown calls prompt the user.

>

3. After each tool call

>

The Context Guardcompresses the result. 3,200 tokens вҶ’ 480 tokens. The Token Budgetdeducts the cost.

>

4. Each step is recorded

>

The Agent Tracelogs the tool name, arguments, result, latency, and tokens. If something goes wrong, you replay the trace.

>

5. Loop detection runs continuously

>

The Loop Detectorwatches for repeated tool calls. If the agent calls read_filethree times with the same args, it injects a nudge.

>

6. Everything is validated

>

The Tool Registryvalidates argument schemas before calls. The Eval Harnessscores outputs against criteria.

>

7. Experiments are tracked

>

The A/B Testercompares model configs. The Prompt Versioningsystem tracks every system prompt change with rollback capability.

>
