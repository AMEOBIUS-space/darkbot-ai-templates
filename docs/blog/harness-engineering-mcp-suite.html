Most MCP servers in the wild have a problem: they pull in dozens of dependencies. Node.js servers need npm install. Python servers need pip install. When you're building an AI agent harness that runs in a sandboxed environment, a CI pipeline, or an air-gapped container, every dependency is a liability — supply chain attack surface, version conflict, broken install.

>

We built a suite of 10 MCP servers that solve this problem. 132 tools. 289 tests. Zero external dependencies.Just Python 3.8+ standard library.

Why Zero Dependencies Matters

>

When you run an AI agent in production, the harness — the scaffolding around the model — determines success or failure. The harness handles context delivery, tool execution, permission checks, observability, and evaluation. Every component is critical.

>

But most MCP-based harness tools assume a friendly environment: network access, package managers, persistent storage. Real production environments break those assumptions:

>
    >
  • CI pipelinesstart from clean containers. Every pip installadds 30+ seconds and a potential failure point.
    • >
  • Sandboxed agentsrun without network access. If your server needs npm installon first run, it's dead on arrival.
    • >
  • Supply chain attackshave compromised npm and PyPI packages multiple times. Fewer dependencies = smaller attack surface.
    • >
  • Minimal containersfor edge deployment can't afford 200MB of node_modules.
    • >
>

Python's standard library is surprisingly capable. json, hashlib, sqlite3, http.server, urllib, re, collections, functools, pathlib— together they cover most of what a harness engineering MCP server needs.

>

How We Did It

>

Transport: JSON-RPC over stdio

>

The MCP protocol uses JSON-RPC 2.0 over stdio. Python's built-in sys.stdin/ sys.stdouthandle this directly — no web framework needed.

>
import sys, json

def handle_request(request):
 method = request.get("method", "")
 params = request.get("params", {})
 if method == "tools/list":
 return {"tools": get_tool_definitions()}
 elif method == "tools/call":
 return execute_tool(params["name"], params.get("arguments", {}))
 return {"error": {"code": -32601, "message": "Method not found"}}

for line in sys.stdin:
 request = json.loads(line)
 response = handle_request(request)
 response["jsonrpc"] = "2.0"
 response["id"] = request.get("id")
 print(json.dumps(response))
 sys.stdout.flush()
>

That's a working MCP server in 15 lines. No imports outside the standard library.

>

Storage: sqlite3 for persistence

>

Several servers need persistent state — trace logs, audit trails, evaluation results. Python's sqlite3module provides a full SQL database with zero setup:

>
import sqlite3

conn = sqlite3.connect(":memory:") # or a file path
conn.execute("""
 CREATE TABLE IF NOT EXISTS traces (
 id TEXT PRIMARY KEY,
 agent_id TEXT,
 event TEXT,
 timestamp REAL,
 tokens_used INTEGER
 )
""")
>

Testing: unittest, not pytest

>

Every server ships with tests using unittest— also in the standard library. The tests run with python -m pytest tests/if pytest is available, but they also run with plain python -m unittest discover tests/in environments where pytest isn't installed.

Getting Started

>

Clone any server and run it:

>
git clone https://github.com/aaameobius-crypto/mcp-context-guard.git
cd mcp-context-guard

# Run tests (no dependencies to install first!)
python -m pytest tests/ -v

# Start the MCP server
python src/server.py
>

Or browse the full suite at harness-engineering-suite.
