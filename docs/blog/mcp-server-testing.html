Testing MCP Servers: A Practical Framework

MCP Testing Python

2026-07-09

Most MCP server tutorials show you how to build a server. Almost none show you how to test it. This matters because an untested MCP server is a liability — one broken tool can stall an agent's entire workflow.

After writing 168 MCP tools across 14 servers (darkbot-ai-templates), here is the testing framework that actually works.

The Three Layers

Layer 1: Unit Tests (Tool Handlers)

Test the handler function directly, bypassing the protocol layer. Fast, deterministic, no I/O:

import unittest
from src.server import hash_text, encode_base64

class TestHashTool(unittest.TestCase):
    """Test the hash_text tool handler in isolation."""
    
    def test_sha256_basic(self):
        result = hash_text({"text": "hello", "algorithm": "sha256"})
        self.assertEqual(result, 
            "2cf24dba5fb0a30e26e83b2ac5b9e29e1b161e5c1fa7425e73043362938b9824")
    
    def test_md5_basic(self):
        result = hash_text({"text": "hello", "algorithm": "md5"})
        self.assertEqual(result, "5d41402abc4b2a76b9719d911017c592")
    
    def test_empty_input(self):
        result = hash_text({"text": "", "algorithm": "sha256"})
        self.assertEqual(len(result), 64)
    
    def test_invalid_algorithm(self):
        with self.assertRaises(ValueError):
            hash_text({"text": "hello", "algorithm": "invalid"})
    
    def test_missing_text(self):
        with self.assertRaises(KeyError):
            hash_text({"algorithm": "sha256"})
    
    def test_unicode(self):
        result = hash_text({"text": "héllo世界", "algorithm": "sha256"})
        self.assertEqual(len(result), 64)


class TestBase64Tool(unittest.TestCase):
    def test_encode(self):
        result = encode_base64({"text": "Hello, World!"})
        self.assertEqual(result, "SGVsbG8sIFdvcmxkIQ==")
    
    def test_encode_unicode(self):
        result = encode_base64({"text": "Привет"})
        self.assertEqual(len(result) > 0, True)
    
    def test_round_trip(self):
        import base64
        original = "Test string 123"
        encoded = encode_base64({"text": original})
        decoded = base64.b64decode(encoded).decode()
        self.assertEqual(decoded, original)
Name tests after the BEHAVIOR, not the implementation. test_sha256_basic describes what it tests. test_1 does not.

Layer 2: Integration Tests (Protocol)

Test that your server correctly handles JSON-RPC requests. This catches serialization bugs, missing fields, and error format issues:

import unittest
import json
from src.server import MCPServer

class TestProtocolLayer(unittest.TestCase):
    """Test JSON-RPC protocol compliance."""
    
    def setUp(self):
        self.server = MCPServer()
        # Register test tools
        self.server.register("echo", "Echo input", {
            "type": "object",
            "properties": {"text": {"type": "string"}},
            "required": ["text"]
        }, lambda args: args["text"])
    
    def _send(self, request: dict) -> dict:
        """Send a JSON-RPC request and get response."""
        return self.server.handle_request(request)
    
    def test_tools_list(self):
        """Server must respond to tools/list with correct schema."""
        resp = self._send({
            "jsonrpc": "2.0",
            "id": 1,
            "method": "tools/list"
        })
        self.assertEqual(resp["jsonrpc"], "2.0")
        self.assertEqual(resp["id"], 1)
        self.assertIn("result", resp)
        self.assertIn("tools", resp["result"])
        
        tool = resp["result"]["tools"][0]
        self.assertEqual(tool["name"], "echo")
        self.assertIn("description", tool)
        self.assertIn("inputSchema", tool)
        self.assertEqual(tool["inputSchema"]["type"], "object")
    
    def test_tools_call_success(self):
        resp = self._send({
            "jsonrpc": "2.0",
            "id": 2,
            "method": "tools/call",
            "params": {"name": "echo", "arguments": {"text": "hello"}}
        })
        self.assertEqual(resp["id"], 2)
        self.assertIn("result", resp)
        self.assertIn("content", resp["result"])
        self.assertEqual(resp["result"]["content"][0]["type"], "text")
        self.assertEqual(resp["result"]["content"][0]["text"], "hello")
    
    def test_tools_call_missing_tool(self):
        resp = self._send({
            "jsonrpc": "2.0",
            "id": 3,
            "method": "tools/call",
            "params": {"name": "nonexistent", "arguments": {}}
        })
        self.assertIn("error", resp)
        self.assertEqual(resp["error"]["code"], -32601)
    
    def test_tools_call_missing_required_arg(self):
        resp = self._send({
            "jsonrpc": "2.0",
            "id": 4,
            "method": "tools/call",
            "params": {"name": "echo", "arguments": {}}
        })
        self.assertIn("error", resp)
    
    def test_unknown_method(self):
        resp = self._send({
            "jsonrpc": "2.0",
            "id": 5,
            "method": "invalid/method"
        })
        self.assertIn("error", resp)
        self.assertEqual(resp["error"]["code"], -32601)
    
    def test_notification_no_response(self):
        """Notifications (no id) should not get a response."""
        resp = self._send({
            "jsonrpc": "2.0",
            "method": "tools/list"
        })
        # Notifications don't get responses
        self.assertNotIn("id", resp)

Layer 3: E2E Tests (Real Pipeline)

Spin up the server as a subprocess and communicate over stdio. This tests the full pipeline including JSON serialization, stdio buffering, and process lifecycle:

import unittest
import subprocess
import json

class TestEndToEnd(unittest.TestCase):
    """Test the server as a real subprocess."""
    
    def setUp(self):
        self.proc = subprocess.Popen(
            ["python3", "src/server.py"],
            stdin=subprocess.PIPE,
            stdout=subprocess.PIPE,
            stderr=subprocess.PIPE,
            text=True
        )
    
    def tearDown(self):
        self.proc.terminate()
        self.proc.wait()
    
    def _rpc(self, request: dict) -> dict:
        """Send request, read response."""
        self.proc.stdin.write(json.dumps(request) + "\n")
        self.proc.stdin.flush()
        line = self.proc.stdout.readline()
        return json.loads(line)
    
    def test_list_tools_e2e(self):
        resp = self._rpc({
            "jsonrpc": "2.0",
            "id": 1,
            "method": "tools/list"
        })
        self.assertEqual(resp["id"], 1)
        self.assertIn("tools", resp["result"])
        self.assertGreater(len(resp["result"]["tools"]), 0)
    
    def test_call_tool_e2e(self):
        # First list tools to get a valid tool name
        resp = self._rpc({"jsonrpc": "2.0", "id": 1, "method": "tools/list"})
        tool_name = resp["result"]["tools"][0]["name"]
        
        # Call it
        resp2 = self._rpc({
            "jsonrpc": "2.0",
            "id": 2,
            "method": "tools/call",
            "params": {"name": tool_name, "arguments": {}}
        })
        self.assertEqual(resp2["id"], 2)
        self.assertIn("result", resp2)
    
    def test_concurrent_calls(self):
        """Server should handle multiple sequential calls."""
        for i in range(5):
            resp = self._rpc({
                "jsonrpc": "2.0",
                "id": i + 1,
                "method": "tools/list"
            })
            self.assertEqual(resp["id"], i + 1)
E2E tests require the server to be runnable as python3 src/server.py. If your server uses environment variables or config files, make sure defaults work in a clean subprocess.

Test Discovery and Organization

Structure your tests so pytest discovers them automatically:

mcp-my-server/
├── src/
│   ├── __init__.py
│   ├── server.py
│   └── tools.py
├── tests/
│   ├── __init__.py
│   ├── test_tools.py        # Layer 1: unit tests
│   ├── test_protocol.py     # Layer 2: integration
│   └── test_e2e.py          # Layer 3: e2e
├── pyproject.toml
└── Makefile

Makefile for convenience:

test:
\tpython -m pytest tests/ -v

test-unit:
\tpython -m pytest tests/test_tools.py -v

test-e2e:
\tpython -m pytest tests/test_e2e.py -v

coverage:
\tpython -m pytest tests/ --cov=src --cov-report=html

Common Testing Pitfalls

1. Testing mocks instead of code

If your test mocks json.loads, you are testing the mock, not your server. Only mock external dependencies (network, filesystem).

2. Not testing error paths

Test what happens when: required args are missing, values are wrong types, the tool handler raises, the client sends malformed JSON, the server receives unknown methods.

3. Testing with real API keys

Never put real API keys in test files. Use environment variables with safe defaults:

def setUp(self):
    self.api_key = os.environ.get("TEST_API_KEY", "test-key-not-real")

4. Not testing tool discovery

If tools/list returns an empty array, the client sees zero tools. Always test that your tools are discoverable.

CI Integration

GitHub Actions workflow for automatic testing:

# .github/workflows/ci.yml
name: CI
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        python: ['3.10', '3.11', '3.12']
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: ${{ matrix.python }}
      - run: pip install pytest pytest-cov
      - run: python -m pytest tests/ -v --tb=short
      - run: python -m pytest tests/ --cov=src --cov-fail-under=80
