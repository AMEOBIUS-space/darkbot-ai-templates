Getting Started with MCP: A Beginner's Guide

>

MCPTutorialPython

The Model Context Protocol (MCP) is how AI agents talk to external tools. Instead of hard-coding API calls into your agent, you expose tools through MCP, and any MCP-compatible client (Claude, Cursor, your own agent) can use them.

>

This guide builds a working MCP server in 10 minutes. No npm. No Node.js. No frameworks. Just Python.

What is MCP?

>

MCP is a protocol — a set of rules for communication. An MCP server exposes tools. An MCP client (like Claude Desktop) calls those tools. The protocol uses JSON-RPC 2.0 over stdio (standard input/output).

>
┌────────┐ JSON-RPC ┌────────┐
│ Client │ ◄───────────────▶ │ Server │
│(Claude) │ stdio (lines) │(Python) │
└────────┘ └────────┘
>

The client sends a request like "list your tools" or "call tool X with args Y". The server responds with JSON.

>

Step 1: Create the Server

>

Create a file called server.py:

>
#!/usr/bin/env python3
"""Minimal MCP server — 3 tools, zero dependencies."""
import sys
import json
import hashlib
import urllib.request

def handle_request(request):
 """Process a single JSON-RPC request."""
 method = request.get("method", "")
 params = request.get("params", {})
 req_id = request.get("id")
 
 if method == "initialize":
 return ok(req_id, {
 "protocolVersion": "2024-11-05",
 "capabilities": {"tools": {}},
 "serverInfo": {"name": "my-first-server", "version": "1.0.0"}
 })
 
 if method == "tools/list":
 return ok(req_id, {"tools": [
 {
 "name": "hash_text",
 "description": "Hash text using SHA-256",
 "inputSchema": {
 "type": "object",
 "properties": {
 "text": {"type": "string", "description": "Text to hash"}
 },
 "required": ["text"]
 }
 },
 {
 "name": "fetch_url",
 "description": "Fetch the content of a URL",
 "inputSchema": {
 "type": "object",
 "properties": {
 "url": {"type": "string", "description": "URL to fetch"}
 },
 "required": ["url"]
 }
 },
 {
 "name": "timestamp",
 "description": "Get current timestamp",
 "inputSchema": {"type": "object", "properties": {}}
 }
 ]})
 
 if method == "tools/call":
 name = params.get("name")
 args = params.get("arguments", {})
 
 if name == "hash_text":
 h = hashlib.sha256(args["text"].encode()).hexdigest()
 return ok(req_id, {"content": [{"type": "text", "text": h}]})
 
 if name == "fetch_url":
 resp = urllib.request.urlopen(args["url"], timeout=10)
 content = resp.read().decode()[:5000]
 return ok(req_id, {"content": [{"type": "text", "text": content}]})
 
 if name == "timestamp":
 from datetime import datetime
 ts = datetime.now().isoformat()
 return ok(req_id, {"content": [{"type": "text", "text": ts}]})
 
 return error(req_id, -32601, f"Unknown method: {method}")

def ok(req_id, result):
 return {"jsonrpc": "2.0", "id": req_id, "result": result}

def error(req_id, code, message):
 return {"jsonrpc": "2.0", "id": req_id, 
 "error": {"code": code, "message": message}}

# Main loop: read lines from stdin, write responses to stdout
if __name__ == "__main__":
 for line in sys.stdin:
 line = line.strip()
 if not line:
 continue
 try:
 request = json.loads(line)
 response = handle_request(request)
 print(json.dumps(response))
 sys.stdout.flush()
 except Exception as e:
 print(json.dumps(error(None, -32603, str(e))))
 sys.stdout.flush()
>

Step 2: Test the Server

>

Test it directly from the terminal:

>
# Initialize
echo '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{}}' | python3 server.py

# List tools
echo '{"jsonrpc":"2.0","id":2,"method":"tools/list","params":{}}' | python3 server.py

# Call hash_text
echo '{"jsonrpc":"2.0","id":3,"method":"tools/call","params":{"name":"hash_text","arguments":{"text":"hello"}}}' | python3 server.py
>

You should see JSON responses with the tool results.

>

Step 3: Connect to Claude Desktop

>

Add your server to Claude's MCP config file:

>

Mac:~/Library/Application Support/Claude/claude_desktop_config.json
>Linux:~/.config/claude/claude_desktop_config.json

>
{
 "mcpServers": {
 "my-tools": {
 "command": "python3",
 "args": ["/absolute/path/to/server.py"]
 }
 }
}
>

Restart Claude Desktop. Your tools now appear in the conversation.

>
Use absolute paths in the config. Relative paths do not work — Claude does not know your working directory.

How It Works

>>>>>>>>
StepWhat happens
1. Client sends initializeServer responds with capabilities
2. Client sends tools/listServer returns tool schemas (JSON Schema)
3. User asks a questionClient (Claude) decides which tool to call
4. Client sends tools/callServer executes handler, returns result
5. Client uses resultClaude incorporates the result into its response
>

Tool Schema Anatomy

>

Each tool is defined by a JSON Schema. The schema tells the client what arguments the tool accepts:

>
{
 "name": "hash_text", // unique identifier
 "description": "Hash text", // what the tool does (LLM reads this)
 "inputSchema": {
 "type": "object",
 "properties": {
 "text": {
 "type": "string",
 "description": "Text to hash" // helps LLM use the tool correctly
 },
 "algorithm": {
 "type": "string",
 "enum": ["sha256", "md5", "sha512"], // constrained choices
 "default": "sha256"
 }
 },
 "required": ["text"] // must always provide "text"
 }
}
>
Write clear descriptions. The LLM uses the description to decide when to call the tool. A vague description like "process data" leads to wrong tool calls.
>

Common Patterns

>

Returning Errors

>
if name == "hash_text":
 text = args.get("text")
 if not text:
 return ok(req_id, {
 "content": [{"type": "text", "text": "Error: 'text' is required"}],
 "isError": true
 })
>

Returning Structured Data

>
# Return JSON that the LLM can parse
return ok(req_id, {
 "content": [{
 "type": "text", 
 "text": json.dumps({"hash": h, "algorithm": "sha256", "length": 64})
 }]
})
>

Stateful Tools

>
# Tools can share state via module-level variables
cache = {}

def handle_fetch(args):
 url = args["url"]
 if url in cache:
 return cache[url]
 result = urllib.request.urlopen(url).read().decode()
 cache[url] = result
 return result
